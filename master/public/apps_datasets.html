Sushi Documentation — Applications and Datasets

In short: a Sushi Application takes as input parameters an existing dataset and generates a new dataset.

The Sushi Application

Key characteristics:

  • represented by a Ruby object, defined in a single file, can be a Ruby or a shell script
  • self-documented
  • writing a Sushi Application requires no additional knowledge about the internals of the sushi_fabric framework
  • can be run on the commandline interface (CLI) or via web interface (GUI)
  • generates a Sushi dataset that is reproducible; remarkably, the Sushi Application is not needed to reproduce the Sushi dataset. This means, Sushi Applicaitons can be updated or removed if obsolete without compromising reproducibility

The full definition is at http://rubydoc.info/gems/sushi_fabric/0.1.9/SushiFabric/SushiApp. There is one central attribute

  • params: This attribute serves the 3 purposes of specifying the parameters, the type of UI element and the default value. Additionally it can hold a description that will be used as help text
and there are two key methods:
  • next_dataset: determines the element of the next dataset given the parameters and the input dataset
  • commands: specifies the shell commands that need to be executed to generate the next dataset
An example application is the WordCountApp.rb

The Sushi dataset

  • fully defined and self-contained on the file system; all elements are stored in a single directory on the file system
  • fully documented: analysis algorithms, options, and versions are fully specified
  • fully reproducible: the executed job scripts and the parameters are stored
  • completely portable: can be copied to a different system and can be further analyzed by a bioinformatician without any requirements on the software environment
  • human-readable and machine-parseable
Elements of a Sushi dataset
  • all data files
  • all job scripts that generated the data files
  • parameter file
  • all logs
  • a DataSet meta-information file that specifies the content of the folder
  • a DataSet meta-information file that specifies the parent dataset
An example dataset of the WordCountApp: Word_count_dataset
